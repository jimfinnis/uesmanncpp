1 UESMANN CPP
2 ===========
3 
4 This is the new C++ implementation of the UESMANN modulatory neural network
5 architecture, based on the original code used in my thesis.
6 See [my University web site for more details and publications](http://users.aber.ac.uk/jcf12/research/uesmann/)
7 or the brief introduction below.
8 
9 ## What is UESMANN?
10 
11 UESMANN is a very simple modification of a standard multilayer perceptron
12 (MLP) with a logistic sigmoid activation function, trained using stochastic
13 gradient descent. The modification consists of a modulatory factor *h*
14 on the weights, such that the weights have their nominal values at *h*=0
15 and double those values at *h*=1. The biases are unmodulated.
16 
17 As such, the network is able to perform multiple functions at different
18 modulator levels, and is typically trained using examples of one
19 function at *h*=0 and another at *h*=1, where it typically performs well.
20 For example, a single network can be trained
21 to perform any possible pairing of binary boolean functions in the same
22 number of network parameters (weights and biases) required for a single
23 such function. It has also been tested in MNIST handwriting recognition
24 and line recognition tasks, and in a homeostatic robot control problem.
25 
26 A rather more complete set of documentation, including a description
27 of the network and a Doxygen docs, can be found at
28 
29 https://jimfinnis.github.io/uesmanncpp/html/index.html
30 
31 ## About the code
32 
33 The code itself is very
34 simplistic, using scalar as opposed to matrix operations and no GPU
35 acceleration. This is to make it as clear as possible, as befits a
36 reference implementation, and also to match the implementation used in the
37 thesis. There are no dependencies on any libraries beyond those found in a
38 standard C++ install, and libboost-test for testing. You may find the code
39 somewhat lacking in modern C++ style because I'm an 80's coder.
40 
41 Implementations of the other network types mentioned in the thesis
42 are also included:
43 
44 * output blending (training two networks with identical architectures
45 to perform the two different functions and using the modulator to
46 linearly interpolate between their outputs);
47 * h-as-input (applying the modulator as an extra input to a standard MLP
48 and training accordingly)
49 * plain (a straightforward MLP with no modifications)
50 
51 
52 
53 I originally intended to use Keras/Tensorflow,
54 but would have been limited to using the low-level Tensorflow operations
55 because of the somewhat peculiar nature of optimisation in UESMANN:
56 we descend the gradient relative to the weights for one function,
57 and the gradient relative to the weights times some constant for the other,
58 alternating between the two. This makes the standard optimisers (such as ADAM)
59 unsuitable. More investigation is planned, however, because
60 a UESMANN layer may prove useful within a larger system.
