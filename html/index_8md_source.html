4 ## Introduction
5 
6 This is the new C++ implementation of UESMANN, based on the original
7 code used in my thesis. That code was written as a ```.angso```
8 library for the Angort language, and has evolved to become
9 rather unwieldly (as well as being intended for use from a language
10 no-one but me uses).
11 
12 The code is very simplistic, using scalar as opposed to matrix operations
13 and no GPU acceleration. This is to make the code as clear as possible,
14 as befits a reference implementation, and also to match the implementation
15 used in the thesis. There are no dependencies on any libraries beyond
16 those found in a standard C++ install, and libboost-test for testing.
17 You may find the code somewhat lacking in modern C++ style because I'm
18 an 80's coder.
19 
20 I originally intended to write use Keras/Tensorflow,
21 but would have been limited to using the low-level Tensorflow operations
22 because of the somewhat peculiar nature of optimisation in UESMANN:
23 we descend the gradient relative to the weights for one function,
24 and the gradient relative to the weights times some constant for the other.
25 A Keras/Tensorflow implementation is planned.
26 
27 Implementations of the other network types mentioned in the thesis
28 are also included.
29 
30 The top level class is @ref Net, which is an virtual type describing the neural net interface
31 and performing some basic operations. Other classes are:
32 
33 * NetFactory, which creates, loads and saves these concrete subclasses of Net:
34  * BPNet, a plain, unmodulated MLP trained with backprop
35  * OutputBlendingNet and HInputNet, which are alternative modulatory
36  networks described below
37  * UESNet, the UESMANN network
38 * ExampleSet, which is a set of examples for training networks
39 * Net::SGDParams, which controls how training is performed (including
40 crossvalidation and hyperparameters)
41 * MNIST, which encapsulates MNIST-format data sets and from which
42 ExampleSet instances can be generated.

45 ## The library and test executables
46 
47 The entire library is include-only, just include
48 netFactory.hpp to get everything. The provided CMakeLists.txt
49 builds two executables:
50 
51 * **uesmann-test** runs a set of test suites written using the Boost test framework.
52 * **genBoolGrid** trains 1000 UESMANN networks for every possible
53 binary boolean function pairing and calculates how many perform
54 the required function (this test was designed to ensure that the
55 library's output matched that from equivalent code used in the thesis).

57 ## The network
58 
59 The network implemented is a modified version of the basic Rumelhart, Hinton
60 and Williams multilayer perceptron with a logistic sigmoid activation
61 function, and is trained using stochastic
62 gradient descent by the back-propagation of errors. The modification
63 consists of a modulatory factor \f$h\f$, which essentially doubles
64 all the weights at 1, while leaving them at their nominal values at 0.
65 Each node has the following function:
66 
67 \f[
68 y = \sigma\left(b+(h+1)\sum_i w_i x_i\right),
69 \f]
70 
71 where
72 
73 * \f$y\f$ is the node output,
74 * \f$\sigma\f$ is the "standard" logistic sigmoid activation \f$\sigma(x) = \frac{1}{1+e^{-x}}\f$
75 * \f$b\f$ is the node bias,
76 * \f$h\f$ is the modulator,
77 * \f$w_i\f$ are the node weights,
78 * \f$x_i\f$ are the node inputs.
79 
80 The network is trained to perform different functions at different modulator
81 levels, currently two different functions at h=0 and h=1.
82 This is done by modifying the back-propagation equations to find the
83 gradient \f$\frac{\partial C}{\partial w_i(1+h) }\f$ (i.e. the
84 cost gradient with respect to the modulated weight). Each example is
85 tagged with the appropriate modulator value, and the network is trained
86 by presenting examples for alternating modulator levels so that the
87 mean gradient followed will find a compromise between the solutions at
88 the two levels.
89 
90 It works surprisingly well, and shows interesting transition behaviour as
91 the modulator changes. In the thesis, it has been tested on:
92 
93 * boolean functions: all possible pairings of binary boolean functions
94 can be performed in the same parameter space as a single boolean function
95 (i.e. two hidden nodes);
96 * image classification: it performs well on line recognition tasks,
97 modulating between horizontal and vertical line recognition in noisy
98 images; and on the MNIST handwriting recognition task, modulating between
99 recognising digits with their nominal labelling and an alternate
100 labelling with a maximal Hamming distance;
101 * robot control: in a homeostatic robot task, a real robot transitioned
102 between exploration and phototaxis as simulated battery charge (which
103 provided the modulator) fell.
104 
105 The performance of UESMANN was tested against:
106 
107 * output blending: performing modulation by training two networks, one
108 for each modulation level, and interpolating between the outputs using
109 the modulator
110 * h-as-input: providing the the modulator h as as extra input to a
111 plain MLP and training accordingly.
112 
113 No enhancements of any kind were used in either UESMANN or the other
114 networks to provide a baseline performance with no confounding factors.
115 This means no regularisation, no adaptive learning rate, not even
116 momentum (Nesterov or otherwise).
